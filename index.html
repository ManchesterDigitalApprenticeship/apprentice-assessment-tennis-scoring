What to do

This application keeps track of players' scores in a game of tennis and determines a winner.

At the moment scores are tracked as a number of points 0, 1, 2, 3, 4 etc for each player, but his isn't quite enough to properly score a game of tennis, and the correct socring rules need to be implemented.

  • Use the application to start and play through a few games of tennis. The buttons on this page increase the score for each player, and the 'New Game' button resets the current game.
  • Read through the code in app/home/tennisService.js. Identify what each function in the file does.
  • Identify the function which returns the score line for the game currently in progress.
  • With your partner, discuss what you will have to do to make the score display as described below.
  • Modify the code so that the game is properly scored.

Scoring tennis

  1. Individual player scores from zero to three points are described as "love", "fifteen", "thirty", and "forty" respectively. e.g. if player1 has 2 points, and player2 0 points, the score would be "thirty - love"
  2. If at least three points have been scored by each player, and the scores are equal, the score is "deuce". e.g. if player1 and player2 have both scored 5, the score is "deuce"
  3. If at least three points have been scored by each player and a player has one more point than his opponent, the score of the game is "advantage" for the player in the lead. e.g. if player1 has scored 4 and player2 3, the score is "advantage player1"
  4. A game is won by the first player to have won at least four points in total and at least two points more than the opponent.
